### 要求

在逐句修改之前，先审阅这一章的段落结构是否合理。你可以建议：

1. **合并段落**：多个段落应该合为一段（例如结论章应精简为一段）
2. **拆分段落**：某段过长，应拆分为多段
3. **删除段落**：某段内容冗余或不应出现在本章
4. **重排段落**：调整段落顺序使逻辑更通顺
5. **无需调整**：结构已经合理

**重要**：你在此阶段做出的结构决策（合并、删除、拆分等）是最终决策，后续的章节审阅不会再推翻这些结构调整。请一次性做出完整、一致的结构规划——如果你认为某些小节应该保留，就不要合并它们；如果你决定合并，后续就不会再要求拆回来。导师修改意见是你最重要的依据，严格遵循导师的明确要求，不要自行添加导师未要求的内容（如导师未提及的小节）。

请以如下 JSON 格式回复：

{{
  "needs_restructure": true或false,
  "feedback": "对当前结构的整体评价",
  "actions": [
    {{
      "type": "merge",
      "paragraphs": [0, 1, 2],
      "instruction": "将这三段合并为一段，保留核心结论，删除冗余展开"
    }},
    {{
      "type": "split",
      "paragraph": 3,
      "instruction": "在第二个论点处拆分为两段"
    }},
    {{
      "type": "delete",
      "paragraph": 4,
      "instruction": "此段与前文重复，删除"
    }},
    {{
      "type": "reorder",
      "from": 2,
      "to": 0,
      "instruction": "这段应该放在最前面作为总领"
    }}
  ]
}}

如果结构合理不需要调整，actions 为空数组。
注意：段落编号从 0 开始。合并操作中 paragraphs 列表里的段落会被合为一段。
