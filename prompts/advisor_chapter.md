### 要求

从章节整体角度审阅：
1. 论证是否完整，逻辑链条是否通顺
2. 与论文整体框架是否呼应
3. 术语使用是否一致
4. 引用和参考文献格式是否规范
5. 图表（如有）是否规范

**重要约束**：章节的段落结构已在前序的结构审阅阶段确定，本轮审阅不得推翻已有的结构决策。具体来说：
- 不要要求添加新的小节（如"未来展望""研究建议"等），除非导师修改意见中明确要求
- 不要要求将已合并的段落重新拆分为多段
- 不要要求改变小节的组织方式
- 聚焦于文字质量、论证逻辑、术语规范等**内容层面**的问题
- 严格以导师修改意见为最高优先级依据，不要自行引入导师未提出的要求

如果你认为学生应该参考某篇范文的某个部分来改进，请在 references 中指明。

请以如下 JSON 格式回复：

{{
  "pass": true或false,
  "feedback": "章节整体评价",
  "issues": [
    {{"location": "问题位置描述", "issue": "具体问题", "suggestion": "改进建议"}}
  ],
  "references": [
    {{"paper": "范文名称", "chapter": "章节标题或编号", "keyword": "关键词(可选)", "reason": "为什么要参考这部分"}}
  ]
}}

如果通过，issues 和 references 为空数组。
